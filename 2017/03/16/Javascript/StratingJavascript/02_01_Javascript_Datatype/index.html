02.1. Primitive Datatype

  • number, string, boolean, null, undefined
  • Javascript 는 느슨한 타입 체크 언어이다.
    • C언어, Java 와는 달리 ‘var’ 라는 한 가지 키워드로만 변수를 선언한다.

02.1.1. Number

  • C : int, long, float, double …
  • Javascript
    • number type 만 존재
      • 정수형이 따로 없고 모든 숫자를 실수로 처리
    • 64bit 부동소수점 형태로 저장 (C 의 double과 유사

02.1.2. String

  • 작은따옴표(‘), 큰따옴표(“) 를 감싸서 생성
  • C : char type 존재
  • Javascript : string 하나만 존재
  • source code 해석
    • 한 번 생성된 문자열은 읽기만 가능, 수정은 불가능하다.

02.1.3. Boolean

  • 값은 true, false 만 존재

02.1.4. null, undefined

  • null, undefined : ‘값이 비어있음’을 뜻함.
    • undefined type
      • 값이 할당되지 않은 변수
      • 타입 - 변수 자체의 값 : undefined type - undefined
    • null type
      • 명시적으로 값이 비어있음을 나타내는 데 사용
      • 타입 - 변수 자체의 값 : null type - object
  • source code 해석
    • nullVar의 typeof 결과가 null이 아니라 object이다.
    • 따라서 일치 연산자(===)를 사용해서 변수의 값을 직접 확인해야 한다.
